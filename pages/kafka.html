History

Apache Kafka was originally developed at LinkedIn by Jay Kreps, Neha Narkhede, and Jun Rao, starting around 2010. It was named after the writer Franz Kafka because it was a system optimized for writing. LinkedIn open-sourced it and donated it to the Apache Software Foundation in 2011. Confluent, the company commercializing Kafka, was founded by its creators in 2014. Kafka now processes 7+ trillion messages per day at LinkedIn alone.

Visual Overview

Why We Use Apache Kafka

Kafka solves the problem of moving high-volume data between systems reliably and in real time. Its log-based, distributed architecture means messages are persisted to disk, replicated for fault tolerance, and can be consumed by multiple systems independently. This makes it the backbone of real-time data pipelines, event-driven microservices, and stream processing.

Applications & Use Cases

  • Real-time log aggregation from distributed services
  • Event-driven microservices communication
  • Stream processing with Kafka Streams or Apache Flink
  • Data pipeline: database → Kafka → data warehouse
  • Fraud detection and real-time alerting
  • Activity tracking (clicks, page views, user events)

Key Facts & Features

  • Topics are divided into partitions for parallel processing
  • Producers write records; Consumers read from offsets they track
  • Replication factor ensures fault tolerance across brokers
  • Retention can be time-based or size-based (data replay is possible)
  • Kafka Connect provides pre-built connectors for 200+ systems
  • Kafka Streams allows stateful stream processing without Spark/Flink
