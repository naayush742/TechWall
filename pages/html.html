01 — Overview

What exactly is HTML?

HTML uses tags — short labels wrapped in angle brackets — to tell the browser what each piece of content actually is: a heading, a paragraph, a link, an image, a form. The browser reads those tags and renders the page accordingly.

  • 🏷️
    Markup, not programming

    HTML doesn't have loops or logic — it describes the structure and meaning of content using nested tags called elements.

  • 🌐
    HyperText = links

    The "HyperText" part means pages can link to other pages — the foundational idea of the entire World Wide Web.

  • 🧠
    Semantic meaning

    <article>, <nav>, <header> — modern HTML tells search engines and screen readers what content actually means, not just how it looks.

  • ⚡
    Works with CSS & JS

    HTML provides the structure, CSS adds the styling, and JavaScript adds the behaviour — three separate, cooperating languages.

03 — History

From a physicist's memo to the web's universal language

HTML's story spans over three decades, a browser war, a standards battle, and the death of Flash — all converging on a single, continuously updated living document.

1989

Tim Berners-Lee's "Vague but exciting" memo

A British physicist at CERN writes a proposal titled "Information Management" to help researchers share documents — his boss labels it "vague, but exciting." HTML and the World Wide Web are born from this one document.

1991

HTML Tags — 18 elements, one document

Berners-Lee publishes "HTML Tags" — a modest CERN document listing just 18 elements. The first website ever (info.cern.ch) goes live on August 6, 1991, explaining what the World Wide Web is.

1993–94

Mosaic, the <img> tag, and a browser war begins

Marc Andreessen and Eric Bina build Mosaic at NCSA — the first browser to display images inline. Andreessen proposes the <img> tag, which Berners-Lee had doubts about. The W3C is founded in 1994 to standardize HTML.

1995–97

HTML 2.0, then 3.2 — the browser wars

Netscape and Internet Explorer race to add proprietary tags — <blink>, <marquee> — while W3C tries to standardize. HTML 3.2 (1997) is the first version formally published as a W3C Recommendation.

1999

HTML 4.01 — the mature, stable version

HTML 4.01 publishes in December 1999, separating structure (HTML) from presentation (CSS). For the next decade it remains the dominant standard, while Flash fills the gap for rich media.

2004–08

WHATWG breaks away, HTML5 begins

Apple, Mozilla, and Opera, frustrated with W3C's push toward XHTML, found the Web Hypertext Application Technology Working Group (WHATWG) in 2004 and begin work on what becomes HTML5.

2010–14

Steve Jobs kills Flash, HTML5 wins

Apple's 2010 letter "Thoughts on Flash" refuses Flash on iPhone, citing HTML5 as the standard for video and rich content. HTML5 is published as a W3C Recommendation in October 2014, with <video>, <canvas>, and semantic elements.

2019–26

The Living Standard — no more version numbers

W3C and WHATWG agree in 2019 that WHATWG is the sole publisher of HTML. The "HTML Living Standard" updates continuously — the last update was June 28, 2026. New elements like <dialog>, <search>, and the Popover API land without ceremony.

04 — Use cases

What HTML actually powers

HTML isn't a niche technology — every category of web product, from a personal blog to an enterprise dashboard, starts with an HTML file.

Websites & blogs

Every article, post & page

Every webpage — news article, blog post, or personal site — is an HTML document rendered by a browser.

E-commerce

Product pages & checkouts

Every "Add to cart" button, product image, and checkout form is an HTML element structured for accessibility and conversion.

Web apps

Single-page applications

React, Vue, and Angular all ultimately output HTML to the browser — JavaScript frameworks build on top of the same HTML foundation.

Email

HTML email newsletters

Every formatted marketing email is an HTML document — email clients render a subset of HTML, often from the early 2000s spec.

Dashboards

Data & analytics UIs

Charts, tables, and interactive controls in business dashboards are HTML elements styled with CSS and powered by JavaScript.

Learning

The universal first lesson

HTML is universally the first technology taught in web development — no setup required, just open a text editor and a browser.

05 — How it works

How a browser turns HTML into a visible page

From receiving a file to painting pixels on screen, the browser runs the same four-stage process every time.

STEP 1

Parse the HTML

The browser reads the HTML file top-to-bottom, building a tree of nodes called the DOM (Document Object Model).

STEP 2

Apply CSS styles

The browser builds a CSSOM from stylesheets, then combines it with the DOM into a "Render Tree" — what to draw and how.

STEP 3

Layout (reflow)

Each element in the render tree is given an exact position and size on the page — left, top, width, height calculated.

STEP 4

Paint to screen

Pixels are painted to the display. JavaScript can then change the DOM, triggering a re-layout and repaint of affected elements.

06 — Versions

HTML through the years

HTML's versioning history reflects the political and technical battles of the web industry — culminating in a single, living specification with no version numbers.

1.0
1991
18 tags. Documents, links, headings.
2.0
1995

3.2
1997
First W3C Recommendation. Applets, scripts.
4.01
1999
Mature standard. Separation from CSS formalized.
XHTML 1.0

Stricter HTML as XML. Eventually abandoned.

Video, canvas, semantics. Flash finally dead.
HTML 5.1/5.2
2016–17
Incremental W3C updates. Last versioned W3C spec.
Living Std
2021 – now
WHATWG sole publisher. Updates daily. No more versions.

Since January 2021, the W3C no longer publishes its own HTML specification. The WHATWG HTML Living Standard — last updated June 28, 2026 — is the single authoritative source for all HTML.
