The checkpoint deciding what gets in, and what gets turned away.

A firewall is a security system — hardware, software, or both — that monitors and controls network traffic based on a defined set of rules. It's the most foundational layer of network defense in existence. This page covers where it came from, how it actually decides what to block, where it's used today, and where to learn it properly.

01 / origin

From a research paper to the front line of every network

The term "firewall" was borrowed from construction — a literal wall built to stop fire from spreading between rooms. Network engineers needed the same idea: stop a threat in one part of a network from spreading into another.

The earliest formal description of network packet filtering was published by Digital Equipment Corporation (DEC) in 1988. By around 1991–92, DEC turned the idea into the first real commercial product, DEC SEAL. These early systems inspected only the basic envelope of a packet — its source and destination addresses and port numbers — to decide whether to let it pass. This approach, now called static packet filtering, treated every single packet as a brand-new, isolated decision, with no memory of the connection it belonged to.

That changed in December 1993, when Israeli company Check Point Software Technologies, founded by Gil Shwed, released FireWall-1 — the first commercially available stateful inspection firewall. Instead of evaluating each packet in isolation, it tracked the state of entire connections, dramatically improving both security and performance. The same year, Trusted Information Systems released the Firewall Toolkit (FWTK), the first open-source firewall, pushing inspection further up to the application layer.

For over a decade, stateful inspection remained the industry standard, with competitors refining the same core model. The next real leap came in July 2007, when Palo Alto Networks introduced the concept of the Next-Generation Firewall (NGFW) — combining traditional filtering with deep packet inspection, intrusion prevention, and application-aware controls in a single device, which remains the dominant architecture in enterprise security today.

02 / mechanics

How a firewall actually decides: allow or deny

Different firewall types inspect traffic at different depths — the deeper the inspection, the more context the firewall has, at the cost of more processing.

Every packet hits the rule table first. A match against an "allow" rule lets it through to the protected network; anything unmatched, or matching a "deny" rule, gets dropped before it ever reaches internal systems.

◆ Rule / ACL

A defined condition — source, destination, port, protocol — paired with an allow or deny action.

◆ Stateful inspection

Tracks the full lifecycle of a connection so only legitimate, expected return traffic is automatically allowed back in.

◆ DMZ (demilitarized zone)

A buffer network segment, isolated by firewalls on both sides, where public-facing servers live separately from internal systems.

◆ Default deny

The security best practice of blocking everything by default and only explicitly allowing known-safe traffic.

◆ Deep packet inspection (DPI)

Examining the actual content of packets, not just headers, to catch threats hidden inside otherwise normal-looking traffic.

◆ NGFW

Combines traditional filtering with intrusion prevention, application identification, and threat intelligence in one system.

03 / applications

Where firewalls are standing guard right now

Firewalls exist at nearly every boundary a network can have — between you and the internet, between departments, even between individual applications.

01

Home and small office routers

Nearly every consumer router includes a basic stateful firewall, silently filtering unsolicited inbound traffic by default.

02

Enterprise network perimeters

Organizations deploy NGFWs at the boundary between their internal network and the public internet as the first line of defense.

03

Cloud infrastructure

Cloud platforms offer software-defined firewalls (security groups, network ACLs) to control traffic between virtual machines and services.

04

Data center segmentation

Internal firewalls separate sensitive zones (databases, payment systems) so a breach in one segment can't easily spread to another.

05

Personal device protection

Operating systems like Windows and macOS ship with built-in host-based firewalls controlling which apps can send or receive traffic.

06

Web application firewalls (WAF)

A specialized firewall type that filters HTTP traffic specifically, blocking attacks like SQL injection before they reach a web app.
