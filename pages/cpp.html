History

C++ was created by Bjarne Stroustrup at Bell Labs starting in 1979, initially called 'C with Classes'. It added object-oriented features to C while maintaining C's performance. The name C++ (using the increment operator) was coined in 1983. It was standardized as C++98, with major updates in C++11, C++14, C++17, C++20, and C++23. C++11 was a landmark modernization with move semantics, lambdas, range-based for loops, and smart pointers.

Why We Use C++

C++ provides the control of C with the abstraction power of OOP and generic programming (templates). This combination — zero-cost abstractions — makes it ideal for software where every nanosecond matters: game engines (Unreal Engine), browsers (Chrome V8, Firefox Gecko), scientific computing, high-frequency trading, and embedded systems with more resources than pure C projects.

Applications & Use Cases

  • Game engines: Unreal Engine, id Tech, CryEngine
  • Browsers: Chrome (V8, Blink), Firefox (Gecko, SpiderMonkey)
  • Operating system components (Windows, macOS internals)
  • High-frequency trading platforms
  • Computer vision libraries (OpenCV)
  • Scientific and numerical computing (ROOT at CERN)

Key Facts & Features

  • RAII (Resource Acquisition Is Initialization): destructors handle cleanup
  • Templates enable generic programming with zero runtime cost
  • STL (Standard Template Library): vector, map, set, algorithm
  • Smart pointers (unique_ptr, shared_ptr) replace manual memory management
  • C++20 added concepts, coroutines, ranges, and modules
  • Compilers: GCC, Clang, MSVC; tools: CMake, vcpkg, Conan
