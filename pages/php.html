01 — Overview

What exactly is PHP?

PHP runs on the server, builds the page using data from a database or elsewhere, and sends only the finished HTML to the visitor's browser — they never see the PHP code itself.

  • 🧩
    Embedded in HTML

    PHP code drops directly inside HTML using <?php ?> tags — mixing logic and markup in the same file.

  • 🖥️
    Server-side execution

    Code runs entirely on the web server before the page is sent — visitors never see or download the actual PHP.

  • 🗄️
    Deep database integration

    Built-in support for MySQL and others makes PHP a natural fit for dynamic, data-driven websites.

  • 🆓
    Free, open source & everywhere

    Nearly every web host supports PHP out of the box, making it one of the cheapest languages to deploy.

02 — History

From a résumé tracker to "PHP: Hypertext Preprocessor"

PHP's creator has said outright he never set out to design a programming language — he just kept "adding the next logical step."

1994

Rasmus Lerdorf builds "Personal Home Page Tools"

A set of small CGI binaries written in C, built simply to track who was viewing Lerdorf's online résumé — not intended as a language at all.

Jun 1995

PHP Tools released to the public

Lerdorf releases the source code publicly, inviting the community to fix bugs and extend it — open source from nearly day one.

1995–96

Becomes "FI," then PHP/FI

A complete rewrite adds Perl-like variables, form handling, and HTML-embedded syntax. By April 1996, combined with database support, it's renamed PHP/FI.

1997–98

PHP 3.0 — a real language at last

Andi Gutmans and Zeev Suraski, frustrated by PHP/FI's limits for an e-commerce project, rewrite the parser entirely and collaborate with Lerdorf — PHP 3 ships as "PHP: Hypertext Preprocessor."

1999–2000

The Zend Engine powers PHP 4

Gutmans and Suraski build the Zend Engine, a faster core that becomes PHP 4 — performance gains that fuel PHP's explosive late-90s/early-2000s growth.

2004

PHP 5 brings real object-oriented programming

A proper OOP model arrives, alongside SQLite support — the same year WordPress launches, eventually becoming PHP's biggest showcase.

2015

PHP 7 — a near 2x performance leap

After the unreleased PHP 6 was abandoned, PHP 7 ships with a new engine roughly doubling performance, plus scalar type hints — a major modernization.

2020–26

PHP 8.x, the PHP Foundation & JIT

PHP 8.0 introduces a JIT compiler, union types, and named arguments; the independent PHP Foundation launches in 2021 to fund core development, backed by JetBrains, Automattic, and Laravel.

03 — Use cases

Who actually builds with PHP

From a beginner's first contact form to Facebook's early-2000s infrastructure, PHP scales across very different project sizes.

Content management

WordPress & Drupal

The world's biggest CMS platforms are all written in PHP — powering roughly 43% of every website on the internet through WordPress alone.

E-commerce

WooCommerce & Magento

The most popular e-commerce plugin (WooCommerce) and platform (Magento) both run on PHP, handling product catalogs and checkout flows.

Enterprise apps

Laravel Business Systems

Companies like Nike, Oracle, HSBC, and SAP run internal tools and platforms built with the Laravel framework.

Large-scale platforms

Wikipedia & Etsy

Wikipedia still runs on PHP today, and Slack's backend began life as PHP/Hack — proof that PHP scales far beyond small sites.

APIs & backends

REST APIs for mobile & web

Frameworks like Laravel and Symfony make PHP a common choice for building the API layer behind mobile and JavaScript front-end apps.

Learning web dev

A starting server-side language

Cheap, near-universal hosting support and a huge body of tutorials make PHP an accessible starting point for new developers.

06 — How it works

From a request to a finished page

Every PHP-powered page follows roughly the same request-response cycle.

STEP 1

Browser sends request

A visitor clicks a link or submits a form, sending an HTTP request to the server for a .php file.

STEP 2

Server runs PHP

The web server hands the file to the PHP interpreter, which executes any code inside <?php ?> tags.

STEP 3

Data is fetched

PHP often queries a MySQL database mid-execution to pull in dynamic content like blog posts or user data.

STEP 4

HTML is returned

PHP code is replaced with its output, and the finished, plain HTML page is sent back to the browser.
