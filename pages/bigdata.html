Big Data and Distributed Computing

This page explores the technologies that enable the processing and analysis of petabyte-scale data across distributed clusters.

Apache Kafka

Originally developed at LinkedIn and open-sourced in 2011, Apache Kafka is a distributed event store and stream-processing platform. It was designed to handle real-time data feeds with high throughput and low latency.

Kafka acts as a central nervous system for data, allowing applications to publish, subscribe to, store, and process streams of events as they occur. It is used by thousands of companies, including Uber, Netflix, and Goldman Sachs, for everything from log aggregation to real-time analytics.

Apache Hadoop

Inspired by Google's GFS and MapReduce papers, Apache Hadoop was created by Doug Cutting and Mike Cafarella in 2006 while at Yahoo. It is an open-source framework that allows for the distributed processing of large data sets across clusters of computers using simple programming models.

Hadoop's core components include HDFS (Hadoop Distributed File System) for scalable storage and MapReduce for parallel processing. It laid the foundation for the Big Data era, enabling organizations to store and analyze vast amounts of unstructured data that were previously unmanageable.
