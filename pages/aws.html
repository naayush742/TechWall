01 — Overview

What exactly is AWS?

Amazon Web Services rents out computing power, storage, databases, and 200+ other building blocks on demand — so a company never has to own a single physical server to run a global product.

  • 🖥️
    Compute (EC2)

    Rent virtual servers by the second — scale from one instance to thousands in minutes.

  • 🗄️
    Storage (S3)

    Store and retrieve any amount of data, from a single file to exabytes, with 99.99% availability.

  • ⚡
    Serverless (Lambda)

    Run code without managing a server at all — pay only for the milliseconds it actually executes.

  • 🤖
    AI & ML (Bedrock, SageMaker)

    Access foundation models like Claude, or build and train custom machine learning models at scale.

02 — History

From Amazon's internal plumbing to the internet's backbone

AWS exists because Amazon's own engineers were wasting time rebuilding the same infrastructure for every new project — so the company decided to productize it.

2000–03

The internal scaling problem

Amazon's own engineering teams keep rebuilding the same storage, compute, and database plumbing for every new feature — sparking the idea of standardized internal "web services."

Mar 2006

S3 launches — the birth of AWS

On March 14, 2006, Amazon S3 launches publicly as a simple, highly durable storage service. Within two months, object counts exceed expectations by 100x.

Aug 2006

EC2 arrives

Amazon Elastic Compute Cloud launches in beta, giving developers on-demand virtual servers — the other half of the foundation AWS is still built on today.

2009–12

Rapid service expansion

RDS (managed databases), Elastic Beanstalk, and DynamoDB all launch, while early customers like Netflix and Airbnb build their entire infrastructure on AWS.

2014

Lambda introduces "serverless"

AWS Lambda launches, letting developers run code triggered by events without provisioning or managing any server — a new computing paradigm.

2015–16

AWS becomes Amazon's profit engine

AWS becomes Amazon's most profitable division and crosses $12 billion in annual revenue, cementing it as the clear market leader in cloud computing.

2018–19

Custom silicon & satellites

AWS announces its own custom ARM-based chips (Graviton) and launches ground stations to talk directly to customer satellites — infrastructure ambitions beyond Earth's data centers.

2023–26

The generative AI era

AWS launches Bedrock, giving developers access to foundation models including Anthropic's Claude, alongside continued investment in Trainium and Inferentia AI chips.

03 — Use cases

Who actually runs on AWS

From a two-person startup to a Fortune 500 bank, the same underlying services scale to fit wildly different needs.

Startups

Launch without buying hardware

A founder can launch a global product on a laptop and a credit card — paying only for what's actually used, scaling up as users arrive.

Streaming

Video delivered at global scale

Streaming services run their entire backend — encoding, recommendations, delivery — on AWS to reach millions of simultaneous viewers.

Enterprise

Regulated industries & banks

Compliance certifications and private networking let banks and healthcare firms move sensitive workloads to the cloud securely.

AI & ML

Training & serving AI models

Bedrock and SageMaker give teams access to foundation models and GPU clusters without owning a single chip.

Learning

Students & certification paths

The Free Tier lets students spin up real infrastructure to learn cloud skills and prep for AWS certification exams.

Research

Science at internet scale

Genomics, astronomy, and climate research projects use S3 and EC2 to process datasets too large for any university's own servers.

05 — Global scale

Why "the cloud" actually means real buildings

AWS isn't abstract — it's physical data centers grouped into Regions, each split into independent Availability Zones for resilience.

06 — How it works

From signup to a running application

The basic loop behind almost every AWS project, whether it's a student's first server or a company's production system.

STEP 1

Pick the services you need

Choose compute (EC2/Lambda), storage (S3), and a database (RDS/DynamoDB) for your application.

STEP 2

Configure access with IAM

Identity and Access Management defines exactly who and what can touch each resource — the security backbone of AWS.

STEP 3

Deploy & run

Launch instances or functions through the Console, CLI, or infrastructure-as-code tools like CloudFormation.

STEP 4

Monitor & scale automatically

CloudWatch tracks usage while Auto Scaling adds or removes capacity automatically as demand changes.

07 — Pricing

Pay only for what you use

AWS has no fixed monthly fee for most services — pricing follows three core principles.

💧 Pay-as-you-go

Billed by the second for compute, by the request for serverless, and by the gigabyte for storage — like a utility bill.

📉 Pay less as you use more

Storage and data-transfer pricing is tiered — the more you use, the lower your per-gigabyte rate becomes.

🤝 Save with commitment

Savings Plans and Reserved Instances cut costs up to 72% in exchange for a 1- or 3-year usage commitment.
