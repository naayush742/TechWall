Binary

Base-2 numeral system used internally by almost all computers.

History

Binary mathematics traces to Gottfried Wilhelm Leibniz (1703), who described a base-2 system using 0 and 1. George Boole formalized Boolean algebra in 1854. Claude Shannon's 1937 MIT master's thesis showed how Boolean logic could be implemented with electrical circuits — the founding insight of digital computing. The first electronic computers (ENIAC, 1945; Manchester Baby, 1948) used binary because electronic circuits naturally represent two states: on and off.

Visual Overview

Why We Use Binary

All computers store and process data as binary because transistors have two stable states: on (1) and off (0). From these two states, all numbers, text (ASCII/Unicode), images (pixel values), audio (PCM samples), and programs (machine code instructions) are encoded. Understanding binary is fundamental to understanding how computers work at the lowest level.

Applications & Use Cases

  • Machine code and assembly language instructions
  • Integer and floating-point number representation (IEEE 754)
  • Character encoding (ASCII: 7-bit, UTF-8: variable width)
  • Color encoding in images (RGB: each channel 0-255 = 8 bits)
  • Networking: IP addresses, subnet masks in binary
  • Bitwise operations in low-level programming and cryptography

Key Facts & Features

  • A bit (binary digit) is the smallest unit of data: 0 or 1
  • 8 bits = 1 byte; can represent 0-255 (unsigned) or -128 to 127 (signed)
  • Hexadecimal (base-16) is a shorthand for binary: 1 hex digit = 4 bits
  • Two's complement: standard method for representing negative integers
  • IEEE 754: 32-bit float = 1 sign bit + 8 exponent bits + 23 mantissa bits
  • XOR, AND, OR, NOT are fundamental bitwise operations in all processors
