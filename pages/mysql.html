01 — Overview

What exactly is MySQL?

MySQL is a relational database management system (RDBMS) — it stores data in structured tables with rows and columns, and lets you query, join, and update that data using SQL.

  • 🗂️
    Relational tables

    Data lives in structured tables with defined columns and types — far more organized than a spreadsheet or flat file.

  • 🔗
    Relationships & joins

    Tables connect via keys, so a single query can pull related data — like customers and their orders — together instantly.

  • 🆓
    Open source

    Released under the GPL, MySQL Community Edition is free to download, use, and modify, with paid editions for enterprise support.

  • ⚙️
    Pluggable storage engines

    InnoDB (the default) handles transactions and relationships; other engines optimize for different workloads.

02 — History

From a Swedish consulting tool to Oracle's flagship open-source database

MySQL began as an internal fix for a data-warehousing problem — then went through two of the biggest acquisitions in tech history while staying free and open source.

1994–95

Built to replace an aging tool

At Swedish company TcX, Michael "Monty" Widenius and David Axmark need a faster alternative to their old UNIREG database tool. The first version of MySQL is released on May 23, 1995.

1998

MySQL AB is founded

Widenius, Axmark, and Allan Larsson found MySQL AB to formally support, license, and develop MySQL as a standalone product.

2000

Goes fully open source under GPL

MySQL AB adopts the GNU General Public License, cementing its commitment to free and open-source software — a decision central to its explosive adoption.

2003

InnoDB brings transactions

MySQL 4.0 adds support for the InnoDB storage engine, giving MySQL full ACID-compliant transactions — a turning point for serious production use.

2008

Sun Microsystems acquires MySQL AB

Sun buys MySQL AB for roughly $1 billion — by then MySQL had grown to 8 million installations and around $100 million in revenue.

2010

Oracle acquires Sun — and MySQL with it

Oracle's acquisition of Sun sparks a "Save MySQL" petition over fears of Oracle limiting it. The same day, Monty Widenius forks the codebase to create MariaDB.

2013–19

Continued evolution under Oracle

MySQL 5.6 and 5.7 add JSON support, better replication, and security improvements, while MySQL 8.0 modernizes the SQL language support and default character sets.

2024–26

MySQL 9.x & HeatWave GenAI

MySQL HeatWave adds in-database generative AI, vector search, and AutoML — letting analytics and AI run directly inside the database, no separate tools required.

03 — Use cases

Who actually runs on MySQL

From a single WordPress blog to Facebook's earliest infrastructure, the same relational core scales across wildly different sizes.

CMS platforms

WordPress, Joomla & Drupal

The majority of websites built on WordPress store every post, comment, and setting in a MySQL database behind the scenes.

E-commerce

Product catalogs & orders

Online stores rely on MySQL's transactional guarantees to keep inventory counts and order records accurate, even under heavy traffic.

Web apps

The "M" in the LAMP stack

Linux, Apache, MySQL, and PHP/Python powered a generation of web applications — and that pattern is still everywhere today.

Mobile backends

User accounts & app data

Mobile apps store user profiles, sessions, and content in MySQL through a backend API, often managed via cloud services like AWS RDS.

Learning SQL

Students & first databases

MySQL's free Community Edition and huge documentation base make it the most common first database students learn SQL on.

Analytics

Real-time analytics with HeatWave

MySQL HeatWave runs transactional and analytical queries in the same database, skipping the need to move data to a separate warehouse.

04 — How it works

What happens inside MySQL, layer by layer

MySQL's architecture splits cleanly into a layer that understands SQL and a swappable layer that actually stores the bytes.

🧠 SQL layer

Connection & authentication
Query parser
Query optimizer
Query execution

💾 Storage engine layer

InnoDB (default — transactions, FK)
MyISAM (legacy — fast reads)
Memory (in-RAM temp tables)
Disk / SSD storage

MySQL's "pluggable storage engine" design means the same SQL commands can run on top of different storage strategies depending on whether you need transactions, speed, or in-memory caching.

STEP 1

Define a table

CREATE TABLE sets the columns, data types, and keys for a new dataset.

STEP 2

Insert data

INSERT INTO adds rows — one customer, one order, one product at a time, or in bulk.

STEP 3

Query & join

SELECT ... JOIN pulls related data from multiple tables in a single request.

STEP 4

Update safely

Transactions ensure an UPDATE either fully completes or fully rolls back — never half-applied.

05 — Editions

Free at its core, paid for enterprise extras

MySQL Community Edition has always been free — Oracle's commercial editions add support, security tooling, and managed cloud hosting.

Community
$0
Free and open source under the GPL. The full relational database engine — what most developers actually run.
Enterprise Edition
Custom
Adds 24/7 Oracle support, advanced security (encryption, masking, auditing), and high-availability tooling for businesses.
HeatWave (cloud)
~$0.35/hr+
Oracle's managed cloud service combining transactions, analytics, and built-in generative AI in one database, billed by usage.

Most developers and small projects never need anything beyond the free Community Edition. Enterprise and HeatWave pricing varies by configuration, region, and usage — check Oracle's official pricing page for current numbers.
