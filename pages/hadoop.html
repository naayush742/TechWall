History

Hadoop was created by Doug Cutting and Mike Cafarella in 2005, inspired by two Google research papers: the Google File System (2003) and MapReduce (2004). The name came from Cutting's son's toy yellow elephant. Yahoo! was the first major adopter, running Hadoop on 1,000 nodes by 2008. Apache Hadoop became a top-level Apache project in 2008. The broader Hadoop ecosystem spawned Hive, HBase, Pig, Spark, and dozens of other tools.

Visual Overview

Why We Use Apache Hadoop

Hadoop enables distributed storage (HDFS) and distributed computation (MapReduce/YARN) across clusters of commodity hardware. Instead of buying expensive specialized machines, organizations can use hundreds of cheap servers to store and process petabytes. This democratized big data analytics for companies that couldn't afford Oracle Exadata or Teradata.

Applications & Use Cases

  • Batch processing of massive log files
  • ETL (Extract, Transform, Load) pipelines
  • Data lake storage and archival
  • Clickstream analysis for e-commerce
  • Genomics and bioinformatics large-scale data processing
  • Financial transaction pattern analysis

Key Facts & Features

  • HDFS: files split into 128MB blocks, replicated 3× across nodes
  • MapReduce: splits computation into Map (filter/sort) and Reduce (aggregate) phases
  • YARN: Yet Another Resource Negotiator — cluster resource management
  • Hive: SQL-like queries on Hadoop data
  • Apache Spark largely replaced MapReduce for in-memory processing
  • Used by Facebook, Yahoo, eBay, Twitter, and many enterprises
